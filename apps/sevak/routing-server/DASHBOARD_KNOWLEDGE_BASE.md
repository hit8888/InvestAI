# Agent-Admin Dashboard Knowledge Base

This document contains comprehensive information about the Meaku Agent-Admin Dashboard to answer user questions conversationally.

## Overview

The Meaku Agent-Admin Dashboard is a comprehensive platform for managing AI-powered conversational agents. It allows you to configure, train, monitor, and optimize your AI agent for customer engagement.

## Core Concepts

### What is an AI Agent?

An AI agent is an intelligent conversational assistant that can interact with website visitors, answer questions, capture leads, and guide users through your product or service.

### Key Components

1. **Agent Configuration** - Set up how your agent behaves and appears
2. **Knowledge Sources (Datasets)** - Training data for your agent
3. **Blocks** - Interactive features visitors can use
4. **Conversations** - Track and manage all interactions
5. **Insights** - Analytics and performance metrics

## Feature Categories

### 1. Conversations Management

**Purpose:** Track and manage all conversations between your agent and visitors.

**Key Features:**

- View all conversation history
- Monitor live/active conversations in real-time
- See conversations assigned to you
- Pin important conversations
- View captured leads
- Track link clicks from conversations
- Export conversation data
- Filter and search conversations

**Use Cases:**

- Review how your agent is performing
- Follow up with leads
- Monitor real-time visitor engagement
- Analyze conversation patterns

### 2. Agent Configuration

#### Datasets (Knowledge Sources)

**Purpose:** Train your agent with information about your product, company, and services.

**Types of Datasets:**

- **Webpages:** Add your website pages via URL. The agent will crawl and learn from them.
- **Documents:** Upload PDF files containing product documentation, FAQs, or other information.
- **Videos:** Upload video files that the agent can reference.
- **Slides:** Upload presentation slides for the agent to learn from.

**Actions Available:**

- Upload new content (URLs, files)
- Edit existing items
- Delete items
- Re-embed items (reprocess for training)
- Bulk operations (select multiple items)
- View processing status
- See statistics (pages processed, frames, last updated)

**Best Practices:**

- Add your main product pages
- Include FAQ pages
- Upload product documentation
- Keep content up to date
- Re-embed after making significant changes

## Common Questions & Answers

### How do I train my agent?

1. Go to Datasets
2. Add webpages via URL or upload documents/PDFs
3. Wait for processing to complete
4. The agent will automatically learn from this content

### How do I change the agent's appearance?

Go to Branding page to:

- Change colors
- Set font style
- Upload logo
- Customize intro message
- Configure agent orb

### How do I add products?

1. Go to Controls
2. Find "Product Description" section
3. Click "Add Product"
4. Enter product name and description
5. Save

### How do I configure the Ask AI block?

1. Go to Blocks
2. Click on "Ask AI" block
3. Configure block visibility
4. Set page visibility rules
5. Configure instructions and CTAs
6. Save changes

### How do I see which questions users ask most?

Go to Insights page and look at "Top Questions Asked by Users" section.

### How do I add a new team member?

1. Go to Settings > Members
2. Click "Add Member"
3. Fill in member details
4. Set permissions
5. Save

### How do I connect Google Calendar?

1. Go to Settings > Calendar
2. Click "Connect Google Calendar"
3. Complete OAuth authentication
4. Configure availability and event types

### What is llms.txt?

llms.txt is a structured file that helps search engines find and cite your best content, improving AEO (Answer Engine Optimization) and GEO (Generative Engine Optimization) performance.

### How do I generate llms.txt?

1. Go to Branding page
2. Find "llms.txt" section
3. Select a data source from dropdown
4. Click "Generate"
5. Wait for generation to complete
6. Click "Download" when ready

### How do I re-embed content?

1. Go to the specific dataset type (webpages, documents, etc.)
2. Select the items you want to re-embed
3. Click "Re-embed" button
4. Wait for processing to complete

### How do I see live conversations?

Go to "Live Chats" in the sidebar or navigate to `/active-conversations`.

### How do I filter insights by date?

On the Insights page, use the date range selector at the top to choose a time period.

### What are blocks?

Blocks are interactive features that appear on your website. They include Ask AI (chat), Book Meeting (scheduling), Video Library, and Summarize.

### How do I configure which pages a block appears on?

1. Go to Blocks
2. Click on the block you want to configure
3. Find "Page Visibility" section
4. Add rules to include/exclude specific pages
5. Save

### How do I add support information?

1. Go to Controls
2. Find "Support" section
3. Check the support methods you want (Website URL, Email, Phone)
4. Enter the information
5. Save

### How do I set agent personality?

1. Go to Controls
2. Find "Personality" section
3. Describe how you want the agent to interact
4. Example: "Act as a helpful and clear product expert who guides users with confidence and empathy"
5. Save

### How do I view leads?

Go to Conversations > Leads or navigate to `/conversations/leads`.

### How do I upload a document?

1. Go to Datasets > Documents
2. Click "Upload Documents"
3. Drag and drop PDF files or click to select
4. Wait for processing

### How do I add a webpage?

1. Go to Datasets > Webpages
2. Click "Upload Webpages"
3. Enter the URL
4. Click "Add"
5. Wait for processing

## Troubleshooting

**Agent not responding correctly:**

- Check if datasets are processed
- Review agent instructions in Controls
- Test in Playground
- Re-embed datasets if content changed

**Block not appearing:**

- Check if block is enabled
- Verify page visibility rules
- Check if you're on the correct page

**Can't upload content:**

- Check file size limits
- Verify file format (PDFs for documents)
- Ensure URLs are accessible

**Insights not showing:**

- Check date range
- Verify you have conversations in that period
- Wait for data to process

## Element Identifiers for Actions

This section documents all unique element IDs (identifiers) that can be used in `click` and `text_change` actions. These IDs are used to target specific buttons, inputs, and interactive elements on the dashboard.

### ID Naming Convention

Element IDs follow this pattern: `{page}-{action}-{element-type}`

- **Page prefix**: Identifies which page/feature area (e.g., `datasets`, `members`, `controls`)
- **Action**: Describes what the element does (e.g., `upload`, `save`, `create`)
- **Element type**: The type of element (e.g., `button`, `input`)

### Using IDs in Actions

When creating actions, use these IDs in the `target` field:

- **For click actions**: Use the button/switch/checkbox ID (e.g., `#datasets-upload-documents-button`)
- **For text_change actions**: Use the input/textarea ID (e.g., `#members-email-input`)

**Important Notes:**

- **All IDs in this documentation are actionable**: Only IDs that can be clicked or have their text changed are included
- IDs can be used with CSS selector syntax: `#element-id`
- IDs can also be used directly: `element-id` (without the `#`)
- Always verify the ID exists before using it in an action
- Some IDs may be dynamic (e.g., include variable parts like source type or index)
- Container/wrapper IDs are NOT included - only interactive elements (buttons, inputs, switches, checkboxes, dropdowns)
- For dynamic IDs, replace the variable parts (e.g., `{index}`, `{type}`, `{id}`) with actual values

### Common Patterns

#### Dynamic IDs

Some IDs include dynamic parts that change based on context:

- `datasets-upload-{sourceType}-button` - The `{sourceType}` part changes based on selected dataset type (webpages, documents, videos, slides)
- `integration-{type}-toggle-button` - The `{type}` part is the lowercase integration type (e.g., `salesforce`, `hubspot`)
- `calendar-{id}-*` - Calendar-specific IDs where `{id}` is the calendar ID (for edit/delete operations)
- `tab-{label}-button` - Tab IDs where `{label}` is the sanitized tab label (e.g., `all-conversations`, `create-calendar`)
- `blocks-{blockId}-*` - Block-specific IDs where `{blockId}` is the numeric block ID
- `blocks-cta-{index}-*` - CTA-specific IDs where `{index}` is the CTA index (0, 1, 2, etc.)
- `controls-prompt-{index}-*` - Prompt-specific IDs where `{index}` is the prompt index in the list (0, 1, 2, etc.)
- `playground-browsing-history-{index}-*` - Browsing history item IDs where `{index}` is the item position (0, 1, 2, etc.)
- `playground-utm-{index}-*` - UTM parameter IDs where `{index}` is the parameter position (0, 1, 2, etc.)
- `table-{pageType}-search-input` - Table search input where `{pageType}` is the page type (e.g., `conversations`, `leads`)
- `table-{pageType}-export-button` - Table export button where `{pageType}` is the page type
- `table-{pageType}-page-size-dropdown` - Table page size dropdown where `{pageType}` is the page type

#### Modal/Dialog IDs

Many forms and actions open in modals/dialogs. When targeting elements in modals:

- Form fields and buttons inside modals use the same naming convention as page elements
- Modal buttons often include context in their IDs (e.g., `members-save-button`, `members-cancel-button`)
- Example: `members-*` IDs are used both on the page (`members-add-member-button`) and in the member modal (`members-save-button`, `members-cancel-button`)

**Note**: Only actionable elements within modals have IDs. Modal containers/wrappers do not have IDs as they are not interactive.

### Best Practices for Using IDs

1. **Always verify ID exists**: Before using an ID in an action, ensure it exists in the current page context
2. **Use descriptive IDs**: The naming convention makes it clear what each element does
3. **Check for dynamic parts**: Some IDs have variable parts that need to be resolved at runtime
4. **Modal context**: When actions target elements in modals, ensure the modal is open first
5. **Page navigation**: Navigate to the correct page before attempting to interact with elements
